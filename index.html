Procedural programming

Specifies a series of well structured steps and procedures in order to compose a program. Its benefits include:

  • Re-usability of pieces code designed as procedures (for example, as libraries)
  • Ease of following the logic of program
  • Maintainability of code

Object oriented programming

OOP is a programming language methodology based around the organization of data into classes and objects.

  • Class: A collection of objects
  • Object: A group of related states and behaviour. It is often a representation of a real world object
  • Attributes: Properties of an object

Back to top

Slide 8

Advantages of OOP

  • Clear modular structure which makes it easier to define abstract data types and hide the implementation details.
  • Easier to fix and maintain code because new objects with small differences can be created quickly.
  • OOP is easily adaptable to other software components e.g. when building a graphical user interface (GUI)

Scripting languages

They are designed to integrate and communicate with other programming languages

A common way to distinguish between other programming languages used to create entire applications is that whereas other languages are compiled first before runtime, scripting languages are interpreted straight from the source code, line by line.

Generally easier to learn and faster to code in than more structured and compiled languages e.g. C++

Markup languages

  • They are languages that annotate text so that the computer can interpret it accordingly.
  • Generally used to create and/or style interfaces.

Back to top

Slide 13

Types of markup

  • Presentational Markup: Used by traditional word processing systems with WYSIWYG.
  • Procedural Markup: Integrated with text to provide text processing instructions to programs.
  • Descriptive Markup: Used to label parts of a document as to how they should be treated e.g. 'cite' and 'caption' tags.
